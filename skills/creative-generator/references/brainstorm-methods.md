## 1. 经典头脑风暴

### 适用场景
- 需要快速生成大量创意
- 问题相对开放
- 团队协作探索

### 核心原则
1. **延迟评判** - 生成阶段不批评
2. **数量优先** - 先追求点子数量
3. **叠加创意** - 使用"是的，而且..."思维
4. **鼓励疯狂** - 疯狂点子往往带来突破

### 执行步骤
1. 明确主题（5分钟）
2. 热身练习（5分钟）
3. 自由生成（20-30分钟）
4. 分类整理（10分钟）
5. 评估筛选（15分钟）

### 示例命令
```bash
python scripts/brainstorm.py --theme "提升用户留存率" --count 15
```

### 输出模板
```
# 头脑风暴报告

## 会话信息
- 主题：[主题]
- 日期：[日期]
- 参与者：[参与者]

## 生成的创意
### 类别1：[类别名]
- 创意1：[描述]
- 创意2：[描述]

### 类别2：[类别名]
- 创意3：[描述]

## Top 3 创意
1. [创意1] - 价值：[核心价值]
2. [创意2] - 价值：[核心价值]
3. [创意3] - 价值：[核心价值]

## 下一步行动
1. [行动1]
2. [行动2]
```

## 2. SCAMPER 变换法

### 适用场景
- 改进现有产品/流程
- 渐进式创新
- 系统化思考改进方向

### 维度说明

| 维度 | 英文 | 问题 | 示例 |
|------|------|------|------|
| **S** | Substitute | 可替换什么？ | 替换材料、组件、人员 |
| **C** | Combine | 可合并什么？ | 组合功能、整合流程 |
| **A** | Adapt | 可调整什么？ | 借鉴其他行业、修改设计 |
| **M** | Modify/Magnify | 可放大什么？ | 增加尺寸、频率、强度 |
| **P** | Put to other uses | 还能做什么？ | 新应用场景、新用户群 |
| **E** | Eliminate | 可删除什么？ | 简化流程、去除冗余 |
| **R** | Reverse/Rearrange | 可重组什么？ | 改变顺序、逆向思维 |

### 示例命令
```bash
python scripts/brainstorm.py --method scamper --target "在线教育平台"
```

### 实例：在线教育平台 SCAMPER

**S (Substitute)**
- 用AI老师替代部分人工答疑
- 用视频替代直播（异步学习）
- 用游戏化替代传统测验

**C (Combine)**
- 学习+社交：学习社区功能
- 课程+认证：学完即获证书
- 直播+回放：灵活学习方式

**A (Adapt)**
- 借鉴Netflix的推荐算法
- 借鉴游戏的成就系统
- 借鉴电商的秒杀模式

**M (Modify/Magnify)**
- 增加互动频率
- 放大个性化程度
- 增强社区氛围

**P (Put to other uses)**
- 企业内训平台
- 知识创作者工具
- 技能认证平台

**E (Eliminate)**
- 去除复杂的课程分类
- 简化购买流程
- 减少无效社交功能

**R (Reverse/Rearrange)**
- 学生先提问，老师再讲课
- 先实践，再学理论
- 反向课程设计

## 3. 六顶思考帽

### 适用场景
- 多角度分析决策
- 避免片面思考
- 团队共识形成

### 帽子含义

| 帽子 | 颜色 | 思考模式 | 关注点 |
|------|------|----------|--------|
| 🎩 白帽 | 白色 | 事实 | 数据、信息、已知事实 |
| 🎩 红帽 | 红色 | 情感 | 直觉、感受、预感 |
| 🎩 黑帽 | 黑色 | 谨慎 | 风险、困难、问题 |
| 🎩 黄帽 | 黄色 | 乐观 | 价值、利益、机会 |
| 🎩 绿帽 | 绿色 | 创意 | 新点子、新方法 |
| 🎩 蓝帽 | 蓝色 | 过程 | 总结、决策、控制 |

### 使用顺序
1. **蓝帽** - 定义问题和流程
2. **白帽** - 收集事实信息
3. **红帽** - 表达情感反应
4. **黄帽** - 探索积极面
5. **黑帽** - 识别风险问题
6. **绿帽** - 提出解决方案
7. **蓝帽** - 总结决策

### 示例命令
```bash
python scripts/brainstorm.py --method hats --decision "是否进入新市场"
```

### 实例：是否推出新产品线

**🎩 白帽（事实）**
- 市场规模：500亿，年增长15%
- 竞品：已有3家主要玩家
- 我们的技术储备：中等
- 预算：需要2000万

**🎩 红帽（情感）**
- 直觉告诉我是好机会
- 但也有些担心竞争
- 感觉团队动力不足

**🎩 黑帽（谨慎）**
- 进入壁垒高
- 需要大量资源
- 可能影响现有业务
- 时间窗口可能关闭

**🎩 黄帽（乐观）**
- 新的增长点
- 技术可以复用
- 品牌可以延伸
- 先发优势

**🎩 绿帽（创意）**
- 可以先做小规模测试
- 合作伙伴模式
- 收购现有玩家
- 差异化定位

**🎩 蓝帽（过程）**
- 结论：值得尝试
- 策略：小规模试点
- 时间：Q2启动
- 负责人：产品部

## 4. 逆向头脑风暴

### 适用场景
- 风险识别与预防
- 问题诊断
- 改进现有方案

### 执行步骤
1. 明确目标
2. 思考如何让目标失败
3. 列出所有失败方法
4. 将每个失败翻转为成功策略
5. 识别隐藏风险
6. 制定预防措施

### 示例命令
```bash
python scripts/brainstorm.py --method reverse --target "提升用户满意度"
```

### 实例：如何提升用户满意度

**步骤1：如何让用户不满意？**
1. 产品经常崩溃
2. 客服响应慢
3. 功能不符合预期
4. 价格太贵
5. 没有个性化推荐
6. 广告太多

**步骤2：翻转策略**
1. 提升稳定性 → 99.9%可用性
2. 加快客服响应 → 5分钟内回复
3. 符合预期 → 用户调研+迭代
4. 优化定价 → 分层定价
5. 个性化 → 推荐算法
6. 减少广告 → 会员免广告

**步骤3：隐藏风险**
- 技术债务积累
- 客服团队压力大
- 调研成本高

**步骤4：预防措施**
- 定期技术重构
- AI客服辅助
- 自动化调研工具

## 5. 5 Whys 根因分析

### 适用场景
- 深度问题探索
- 根本原因识别
- 避免治标不治本

### 执行步骤
1. 描述问题
2. 问第一个"为什么"
3. 针对答案再问"为什么"
4. 重复5次
5. 找到根本原因
6. 提出针对性解决方案

### 示例命令
```bash
python scripts/brainstorm.py --method 5whys --problem "用户流失率高"
```

### 实例：用户流失率高

**问题**：用户流失率高达30%

**为什么1**：用户活跃度下降
  ↓
**为什么2**：用户找不到有价值的内容
  ↓
**为什么3**：推荐算法不够精准
  ↓
**为什么4**：缺少用户行为数据
  ↓
**为什么5**：新用户引导流程不完善，无法收集偏好

**根本原因**：新用户引导流程不完善

**解决方案**：
1. 优化引导流程，收集用户偏好
2. 改进推荐算法
3. 提升内容质量
4. 增加活跃用户激励机制

## 6. 星爆法 (Starbursting)

### 适用场景
- 全面探索问题
- 新产品/项目启动
- 识别盲区

### 6个核心问题

| 问题 | 思考方向 |
|------|----------|
| **Who** | 谁？涉及谁？谁受益？谁执行？ |
| **What** | 什么？核心是什么？包含什么？ |
| **Where** | 哪里？在什么场景？什么地方？ |
| **When** | 何时？什么时机？多长时间？ |
| **Why** | 为什么？目的是什么？价值是？ |
| **How** | 如何？怎么实现？用什么方法？ |

### 示例命令
```bash
python scripts/brainstorm.py --method starburst --theme "新产品发布"
```

### 实例：新产品发布

**Who（谁）**
- 目标用户：18-35岁城市白领
- 受益者：用户、公司、合作伙伴
- 执行者：产品、市场、销售

**What（什么）**
- 核心功能：智能推荐
- 包含：App、小程序、Web
- 不包含：线下服务

**Where（哪里）**
- 场景：通勤、工作间隙、睡前
- 地区：一二线城市
- 平台：iOS、Android

**When（何时）**
- 发布时间：Q2
- 最佳时机：节假日
- 迭代周期：2周

**Why（为什么）**
- 目的：解决信息过载
- 价值：节省用户时间
- 差异化：AI驱动

**How（如何）**
- 技术：机器学习
- 推广：内容营销+KOL
- 运营：社区驱动

## 7. 约束释放法

### 适用场景
- 突破思维限制
- 激发创新想法
- 长期规划

### 执行步骤
1. 列出当前约束
2. 假设约束不存在
3. 生成"无约束"点子
4. 识别可落地方案
5. 制定实施路径

### 示例命令
```bash
python scripts/brainstorm.py --method constraint --problem "教育公平" --constraints "预算、师资、地域"
```

### 实例：如何提升教育公平

**当前约束**
- 预算有限
- 师资不足
- 地域限制

**无约束思考**

1. **预算无限制**
   - 为每个学生配备AI导师
   - 建设虚拟现实教室
   - 免费提供所有教材

2. **时间不是问题**
   - 个性化学习路径
   - 终身学习体系
   - 深度技能培养

3. **有任何技术**
   - 脑机接口学习
   - 全息投影教学
   - 实时翻译

**可落地方案**
1. AI辅助教学（现有技术可行）
2. 在线课程平台（成本可控）
3. 混合式教学（线上线下结合）

**实施路径**
- 短期：搭建在线平台
- 中期：引入AI助教
- 长期：VR/AR教室

## 8. 脑力写作 (Brainwriting)

### 适用场景
- 团队协作
- 避免强势声音主导
- 收集多元化想法

### 执行流程
```
第1轮：
  A → 写3个点子 → 传给B
  B → 写3个点子 → 传给C
  C → 写3个点子 → 传给A

第2轮：
  A → 基于B的点子扩展 → 传给B
  B → 基于C的点子扩展 → 传给C
  C → 基于A的点子扩展 → 传给A

第3轮：
  ...重复...

最后：
  收集所有点子 → 分类整理 → 投票筛选
```

### 关键原则
1. 每轮时间：5分钟
2. 安静书写，不讨论
3. 基于他人点子扩展
4. 不批评，不修改他人内容
5. 最后统一筛选

## 9. SWOT 分析

### 适用场景
- 战略评估
- 项目决策
- 竞争分析

### 四个维度

| 维度 | 含义 | 内外部 | 关注点 |
|------|------|--------|--------|
| **S** | Strengths（优势） | 内部 | 核心竞争力 |
| **W** | Weaknesses（劣势） | 内部 | 需要改进 |
| **O** | Opportunities（机会） | 外部 | 市场机遇 |
| **T** | Threats（威胁） | 外部 | 外部挑战 |

### 输出模板
```
# SWOT 分析报告

## 主题
[产品/项目/决策]

## Strengths（优势）
- [优势1]
- [优势2]
- [优势3]

## Weaknesses（劣势）
- [劣势1]
- [劣势2]
- [劣势3]

## Opportunities（机会）
- [机会1]
- [机会2]
- [机会3]

## Threats（威胁）
- [威胁1]
- [威胁2]
- [威胁3]

## 战略建议
1. SO策略（利用优势抓住机会）
2. WO策略（改进劣势抓住机会）
3. ST策略（利用优势应对威胁）
4. WT策略（减少劣势回避威胁）
```

## 10. 方法选择指南

### 按目标选择

| 目标 | 推荐方法 | 原因 |
|------|---------|------|
| 快速生成点子 | 经典头脑风暴 | 数量优先 |
| 改进现有产品 | SCAMPER | 系统化改进 |
| 决策分析 | 六顶思考帽 | 多角度思考 |
| 风险识别 | 逆向头脑风暴 | 翻转思维 |
| 问题诊断 | 5 Whys | 深度分析 |
| 全面探索 | 星爆法 | 6W全覆盖 |
| 突破限制 | 约束释放法 | 打破思维框 |
| 团队协作 | 脑力写作 | 避免主导 |
| 战略评估 | SWOT | 内外分析 |

### 按场景选择

| 场景 | 推荐方法组合 |
|------|-------------|
| 产品创新 | SCAMPER → 六顶思考帽 → 评估矩阵 |
| 问题解决 | 5 Whys → 逆向头脑风暴 → 解决方案 |
| 战略规划 | SWOT → 三层地平线 → Stage-Gate |
| 项目启动 | 星爆法 → 经典头脑风暴 → 优先级排序 |
| 风险管理 | 逆向头脑风暴 → 黑帽思考 → 预防措施 |

---

> 💡 **提示**：多种方法组合使用效果更佳。先发散（生成点子），再收敛（评估筛选）。
