## Layer 1: 创意生成技术库

### 经典头脑风暴

```bash
# 快速生成（适合快速探索）
gemini -m pro -o text -e "" "快速头脑风暴（每个点子1句话）：

主题：[你的主题]

生成10个创意点子，无需解释。"
```

```bash
# 深度生成（适合复杂问题）
gemini -m pro -o text -e "" "深度头脑风暴：

主题：[主题]
背景：[相关背景]
约束：[限制条件]

要求：
- 混合常规与非常规点子
- 考虑不同用户视角
- 包含至少2个'疯狂点子'

每个点子包含：
- 简要描述
- 核心价值
- 主要挑战"
```

### SCAMPER 变换法

**适用**：改进现有产品/流程

```bash
gemini -m pro -o text -e "" "SCAMPER 分析：

目标：[产品/功能/流程]

- S (Substitute)：可替换什么？
- C (Combine)：可合并什么？
- A (Adapt)：可调整什么？
- M (Modify/Magnify)：可放大什么？
- P (Put to other uses)：还能做什么？
- E (Eliminate)：可删除什么？
- R (Reverse/Rearrange)：可重组什么？

为每个维度生成2-3个创新点子。"
```

### 六顶思考帽

**适用**：多角度分析决策

```bash
gemini -m pro -o text -e "" "六顶思考帽分析：

决策：[你正在考虑的决策]

🎩 白帽（事实）：我们知道什么？
🎩 红帽（情感）：直觉反应是什么？
🎩 黑帽（谨慎）：可能出什么问题？
🎩 黄帽（乐观）：有什么好处？
🎩 绿帽（创意）：有什么替代方案？
🎩 蓝帽（过程）：最佳方法是什么？

提供完整的多角度分析。"
```

### 逆向头脑风暴

**适用**：风险识别与预防

```bash
gemini -m pro -o text -e "" "逆向头脑风暴：

目标：[你的目标]

1. 如何让这个目标失败？（列出5-10个方法）
2. 将每个失败方法翻转为成功策略
3. 识别隐藏风险
4. 制定预防措施"
```

### 5 Whys 根因分析

**适用**：深度问题探索

```bash
gemini -m pro -o text -e "" "5 Whys 根因分析：

问题：[描述问题]

连续问5次'为什么'，直到找到根本原因：

为什么1：[问题发生的原因？]
  ↓
为什么2：[原因1发生的原因？]
  ↓
为什么3：[原因2发生的原因？]
  ↓
为什么4：[原因3发生的原因？]
  ↓
为什么5：[根本原因？]

然后提出针对性解决方案。"
```

### 星爆法 (Starbursting)

**适用**：全面探索问题

```bash
gemini -m pro -o text -e "" "星爆法探索：

主题：[你的主题]

回答6个核心问题：
- Who：谁？涉及谁？谁受益？
- What：什么？核心是什么？
- Where：哪里？在什么场景？
- When：何时？什么时机？
- Why：为什么？目的是什么？
- How：如何？怎么实现？

每个问题深入展开3个子问题。"
```

### 约束释放法

**适用**：突破思维限制

```bash
gemini -m pro -o text -e "" "无约束头脑风暴：

问题：[你的问题]
当前约束：[列出约束]

思考：
1. 预算无限制时会怎样？
2. 时间不是问题时会怎样？
3. 有任何技术时会怎样？
4. 没有历史包袱时会怎样？

然后：哪些点子可以缩小规模到现实？"
```

### 脑力写作 (Brainwriting)

**适用**：团队协作，避免强势声音

```
流程：
1. 每人安静写下3个点子（5分钟）
2. 传递给下一个人
3. 基于上一个人的点子扩展（5分钟）
4. 重复3轮
5. 收集所有点子进行筛选
```

### SWOT 分析

**适用**：战略评估

```bash
gemini -m pro -o text -e "" "SWOT 分析：

主题：[你的产品/项目/决策]

- Strengths（优势）：内部强项
- Weaknesses（劣势）：内部弱项
- Opportunities（机会）：外部机遇
- Threats（威胁）：外部威胁

基于SWOT提出战略建议。"
```

## Layer 2: 创意结构化工作流

### 设计思维流程

```
┌─────────────────────────────────────────────────────────────┐
│                    Design Thinking 流程                       │
├─────────────────────────────────────────────────────────────┤
│  1. 同理心 (Empathize)                                       │
│     → 用户研究、观察、访谈、旅程映射、人物画像                   │
│                                                              │
│  2. 定义 (Define)                                            │
│     → 问题框架、观点陈述、HMW问题、设计原则                     │
│                                                              │
│  3. 创意 (Ideate)                                            │
│     → 头脑风暴、思维导图、类比、概念选择                        │
│                                                              │
│  4. 原型 (Prototype)                                         │
│     → 低保真模型、纸面原型、可点击线框图、MVP                   │
│                                                              │
│  5. 测试 (Test)                                              │
│     → 用户测试、反馈收集、迭代、验证                           │
└─────────────────────────────────────────────────────────────┘
```

### 创意工作流命令

```bash
# 完整创意流程
gemini -m pro -o text -e "" "创意流程：

目标：[你的目标]

执行以下步骤：
1. 同理心：描述目标用户画像和痛点

```

### 问题分解框架

```bash
执行：
1. 识别子问题

3. 映射各部分依赖关系

```

### 创意评估矩阵

```bash
gemini -m pro -o text -e "" "创意评估：

2. [点子2]
3. [点子3]
```

```
┌─────────────────────────────────────────────────────────────┐
```

### 创新类型分类

| 类型 | 描述 | 示例 |
|------|------|------|
| **渐进式** | 现有改进 | 产品升级 |
| **邻近式** | 公司新领域 | 新市场进入 |
| **颠覆式** | 行业新范式 | 商业模式创新 |
| **激进式** | 世界首创 | 突破性技术 |

### Stage-Gate 流程

```
阶段-门径模型：

门径1：创意筛选 → 阶段1：范围界定
门径2：二次筛选 → 阶段2：商业论证
门径3：开发决策 → 阶段3：开发
门径4：测试决策 → 阶段4：测试验证
门径5：发布决策 → 阶段5：发布

关键门径：
- Go/No-go 决策点
- 基于标准评估
- 资源分配
- 风险评估
```

## 最佳实践

### 创意生成原则

1. **数量优先** - 先生成大量点子再评判
2. **延后评判** - 生成阶段不批评
3. **叠加创意** - 使用"是的，而且..."思维
4. **拥抱疯狂** - 疯狂点子往往带来实践突破
5. **可视化** - 绘图或图表化创意
6. **设置约束** - 矛盾地，限制反而激发创意
7. **适当休息** - 让创意孵化
8. **混合输入** - 结合不同视角
