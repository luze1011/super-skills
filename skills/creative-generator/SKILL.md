## Layer 1: 创意生成技术库

### 核心方法（详见 references/brainstorm-methods.md）

1. **经典头脑风暴** - 快速生成大量点子
2. **SCAMPER 变换法** - 改进现有产品/流程
3. **六顶思考帽** - 多角度分析决策
4. **逆向头脑风暴** - 风险识别与预防
5. **5 Whys 根因分析** - 深度问题探索
6. **星爆法 (Starbursting)** - 全面探索问题
7. **约束释放法** - 突破思维限制
8. **脑力写作** - 团队协作

### 快速使用

```bash
# 快速头脑风暴
python scripts/brainstorm.py --theme "你的主题" --count 10

# SCAMPER 分析
python scripts/brainstorm.py --method scamper --target "产品名称"

# 六顶思考帽
python scripts/brainstorm.py --method hats --decision "决策内容"
```

## Layer 2: 创意结构化工作流

### 设计思维流程

```
1. 同理心 (Empathize) → 用户研究、观察、访谈
2. 定义 (Define)      → 问题框架、HMW问题
3. 创意 (Ideate)      → 头脑风暴、思维导图
4. 原型 (Prototype)   → MVP、低保真模型
5. 测试 (Test)        → 用户测试、反馈收集
```

### 创意评估

```bash
# 评估创意
python scripts/idea-evaluate.py --ideas "点子1,点子2,点子3"
```

评估维度（1-5分）：
- 可行性：技术上能实现吗？
- 影响力：能带来多大价值？
- 工作量：需要多少努力？（越低越好）
- 风险：失败可能性？（越低越好）

## Layer 3: 创新管理框架

### 三层创新地平线

详见 `references/innovation-frameworks.md`

| 地平线 | 资源 | 时间 | 风险 | 回报 |
|--------|------|------|------|------|
| H1 - 核心 | 70% | 0-2年 | 低 | 已知 |
| H2 - 邻近 | 20% | 2-5年 | 中 | 涌现 |

### Stage-Gate 流程

```
门径1：创意筛选 → 阶段1：范围界定
门径2：二次筛选 → 阶段2：商业论证
门径3：开发决策 → 阶段3：开发
门径4：测试决策 → 阶段4：测试验证
门径5：发布决策 → 阶段5：发布
```

## 最佳实践

1. **数量优先** - 先生成大量点子再评判
2. **延后评判** - 生成阶段不批评
3. **叠加创意** - 使用"是的，而且..."思维
4. **拥抱疯狂** - 疯狂点子往往带来实践突破
5. **设置约束** - 限制反而激发创意
6. **适当休息** - 让创意孵化
